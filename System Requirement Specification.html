1. Introduction

1.1 Purpose

In this modern era of screens and apps, money receipts have become a persistent token of the past. However, taking advantage of the modern-day technology, an easy and seamless solution to keep tabs on the day-to-day spendings and savings can be of utmost convenience. This is where the product, Koto – Finance Manager (v0.0) comes in. It is designed to keep an eye (or a camera) on the receipts so that the user(s) of this product can easily capture the receipt(s) and keep all the logs and bills safely stored in the cloud. The implementation of image recognition technology broadens the scope to store the expense details and run analyses on the transaction data. This allows the user to plan for savings and keep a tab on the income vs expenditure details. Succinctly, the product takes care of the daily monetary issues with just a snap!

2. Overall Description

This section deals with a broad overview of the product Koto – Finance Manager, explaining the functionalities and goals that the software aims to achieve. It also provides details on the dependencies, constraints, operating environment of the product along with user classes and user manuals.

2.1 Product Perspective

Koto - Finance Manager is a smart and easy solution to track the daily life expenditures of any authenticated user of the application. When released, the final product would be the first version of the software. It is designed to store the user’s money receipts (printed or handwritten) or manually entered expenditure(s) to keep the record of the expenses in order to provide an analysis of income, expense and savings at the end of a certain time frame.

2.2 Product Functions

The product functions are heavily user dependent (as elaborated in System Features section). The admin’s function is to only maintain the software and update it whenever necessary. In order to make the functions comprehensible, a use case diagram of the user (actor) is demonstrated below:

Figure: Actor(user) and the Use-cases

2.3 User Classes and Characteristics

The software will consist of mainly two types of users: registered users and the admin. The admin’s job is pretty simple and straightforward, they can only update the software. Since the product keeps track of the user’s personal expenses and income, due to security issues the users data shall not be accessed by the admin. The users can scan the receipts, input manual expenditures, input their income and saving targets and view the ‘Wallet’ for an insight of their financial state.

2.4 Operating Environments

The operating environment of Koto - Finance Manager is listed as follows:

  • Distributed Database
  • Client/Server System
  • Operating System: any system that supports a modern web browser
  • Database Configuration: Django and Google Firebase
  • Platform: Web and Android

2.5 Design and Implementation Constraints

The mobile application of the product is currently designed for android users only. To store the receipt snaps, the product makes use of the Google Cloud and Firebase (the Cloud Storage for Firebase is tightly integrated with Google Cloud Platform). These services are currently free but it comes with usage limitations that include 1000 processes a month, 100 simultaneous database connections and 1 gigabyte of database storage and therefore the scalability issue of the software is involved. Another constraint lies in the fact that the product is heavily depended on the Google’s Image Recognition technology and hence requires an Internet connection for it to work.

2.6 User Documentation

The product will provide users with in-app slideshow instructions to run the users through the basics and a short video tutorial demonstrating the final software product.

2.7 Assumptions and Dependencies

The user of the system requires a smart-phone with an Android Operating System of version as early as 5.0 Lollipop with a built in camera which will be accessed via the app to take pictures. The user should have an internet connection to upload pictures to the cloud and for the image recognition algorithm to return the textual data that has been read from the receipts. For the image recognition algorithm, this product depends on Google’s Vision API. It is assumed that the user(s) of this software is registered via Google account and wants to keep track of their financial status.


3. External Interface

3.1 User Interfaces

The android app will consist of 3 main horizontal tabs at the bottom that the user can swipe through. From left to right, they are - Camera, Wallet and Logs.

Figure: UI Mockup of the Koto Android app

The camera tab will be where users take snaps of the receipts/bills that they want to digitize. It simply has a camera view, a capture button along with an option for flash and a gallery button to pick a previously taken image from the phone's storage to analyze. The user will then be asked to point out keywords and classify the entry.

The home tab is a graphical overview of the user’s expenditure and savings. It will feature a meter representing the user's wallet initially set by their income. All expenditures count towards decreasing the wallet meter like a video game life bar. Leftover money is displayed as savings.

The log tab is where all the user's previously snapped or manually input expenditures are stored in a searchable and chronological manner. The list will have the title key word of the bill along with a date and time. A search button is where users can search according to key words or bill content or date.

As for the product family style guide, Google's Material Design is to be followed as much as possible with layout constraints typical of modern android apps similar to Instagram or Snapchat for example. Software components required for such an app would be an Android Operating System of version as early as 5.0 Lollipop.

3.2 Hardware Interfaces

The primary hardware Koto requires at this point is a smartphone with a built in camera. The device should be able to activate the camera from within the app to take pictures. The secondary requirement is an active Internet connection, either by mobile data or WiFi. A connection capable of uploading pictures and receiving textual data is sufficient. This is to communicate with the Google cloud services which will run the image analysis.

3.3 Software Interfaces

The Koto app will run on a smartphone device running at least Android version 5.0 Lollipop, while a browser-based web interface will run on any computer (Windows, Mac or Linux) with a capable modern web browser. Google’s Cloud Platform will be used to take advantage of its Firebase Backend Service and Google Vision for optical character recognition. The database will be constructed in Python 3 Django version 2. Google’s Cloud services like Firebase, Cloud Storage and use of Vision OCR are commercial components and costs around $3 per month.

Communications Interfaces

The application will send images to Google’s Vision servers to analyze and will receive text data that has been recognized in the image in a format readable and presentable by the app. Here approximately 4MB of data is transferred (dependant largely on the quality of the image). All this occurs via HTTP requests and the images and data transferred are encrypted by Google’s proprietary technology.

    5. Other Nonfunctional Requirements

    Performance Requirements

    Any Interface between a user and software shall have reasonable response time based on Intranet connection

    Safety Requirements

    To ensure no data is lost in case the user decides to change devices or if the user's device is extensively damaged all data will be stored in google cloud storages. This will allow safe storage and easy accessibility from anywhere.

    Security Requirements

    The System shall not disclose any personal information about the users. The Application shall not grant access to an unauthorized user and the Application shall not communicate with any other devices or servers while in use by the user.
    In addition to that users must refrain from uploading any sensitive information such as credit card info, id or any personal information that is not required.

    Software Quality Attributes

    The software quality attributes of koto - finance manager is listed below:

    • AVAILABILITY : As the system will be hosted in cloud servers it will be available all the time for users.
    • CORRECTNESS: The system will pull appropriate information from the receipt images and use Google’s Vision API to provide accurate information based on the users request
    • MAINTAINABILITY: The system will store and maintain all uploaded copies of the receipts.

    Business Rules

    The system will have two different editions based on the business models. The editions will have certain features differences in addition to sharing the common features e.g. scanning receipts, monthly/weekly track records. The two business models are briefly discussed below :

    • Freemium : This version of the app which will be available for the general masses will contain all the basic features described above. This specific version will also contain advertisement in forms of banners or short video clips. User will be able to remove advertisement buy purchasing the add free subscription which will be offered after successful registration to the app.
    • Enterprise edition : This version will be designed specifically for small or large scale businesses and will be subscription based. In addition to the basic features the version will provide extra features such as multiple user access, finance analytics etc.

      Appendix : Glossary

    • API: Application Protocol Interface. This is the part of a program that lets other programs or services interact with the data in the former and viceversa.
